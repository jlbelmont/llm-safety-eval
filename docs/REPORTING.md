# Report Generation Workflow\n\n## Structure\n- `reports/templates/` — Jinja2 LaTeX templates (keep tracked)\n- `reports/sections/` — per-section `.tex` content (keep tracked)\n- `reports/report_config.yaml` — metadata, section order, template refs (keep tracked)\n- `reports/assets/` — figures and tables you reference in the report\n- `reports/build/` — generated `.tex` and PDFs (git-ignored)\n\n## Generate a report\n```bash\nsource .venv/bin/activate\npython scripts/generate_report.py \\\n  --config reports/report_config.yaml \\\n  --output reports/build/research_report.tex\n```\n\n## Build PDF (latexmk)\n```bash\nlatexmk -pdf -output-directory=reports/build reports/build/research_report.tex\n```\n\n## Customization\n- Edit `reports/report_config.yaml` for title/authors/section order/template.\n- Edit section files under `reports/sections/*.tex`.\n- Add figures/tables under `reports/assets/` and reference them in sections.\n- Bibliography: put entries in `reports/refs.bib`; path configured in config.\n\n## Notes\n- Templates/sections/config are tracked; generated PDFs/aux/logs are ignored via `.gitignore`.\n- If `latexmk` is missing, install a TeX distribution (e.g., MacTeX/TeX Live) or build externally.\n*** End Patch ***!
